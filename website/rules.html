Rules

Official competition rules and policies. Please read carefully before submitting.

1. Eligibility

  • The competition is open to individuals and teams worldwide.
  • There is no minimum or maximum team size, though we recommend 1–5 members.
  • Participants may be affiliated with academic institutions, industry organizations, or be independent researchers.
  • Organizers and their direct collaborators may participate for benchmarking purposes but are not eligible for prizes.
  • Participants must disclose any conflicts of interest with organizing institutions (BMW Group, Augsburg University, Anthropic, Cerebras, OpenStream.ai, NTU Singapore, TU Darmstadt, Stanford University).
  • Each individual may be a member of only one team.

2. Registration

  • Teams must register before their first submission to the official leaderboard.
  • Registration requires: team name, member names and affiliations, contact email, and track selection (Open, Cerebras, or both).
  • Registration is open from April 2026 through July 1, 2026. Late registrations may be accepted at organizer discretion but will not be eligible for the mid-phase evaluation.
  • Teams may update their membership up to 7 days before the final evaluation deadline.

3. Allowed Approaches

We adopt a permissive stance. Any approach is permitted unless explicitly prohibited below. This includes but is not limited to:

  • Any LLM (proprietary or open-source, any size)
  • Prompting strategies (system prompts, few-shot, chain-of-thought, self-reflection)
  • Agentic scaffolding (planning-execution separation, retry logic, verification)
  • Multi-agent systems (subagents, orchestration, ensemble methods)
  • Fine-tuning on the provided training data (SFT, RLHF, DPO, etc.)
  • Retrieval-augmented generation (RAG) over environment data
  • External knowledge bases or pre-computed lookup tables (provided they do not encode test set answers)

Cerebras Track: Agents must run on Cerebras-provided inference infrastructure. You may use any model available on the Cerebras platform.

4. Prohibited Behaviors

The following are strictly prohibited and will result in disqualification:

  • Hard-coding answers: Encoding specific test-set answers or lookup tables that map task descriptions to solutions.
  • Exploiting evaluation infrastructure: Reverse-engineering, probing, or exploiting vulnerabilities in the evaluation system, Docker environment, or GitHub Actions pipeline.
  • Probing the hidden test set: Any attempt to extract, infer, or reconstruct hidden test set tasks through the evaluation API or any other channel.
  • Sharing accounts: Sharing registration credentials or API keys across teams.
  • Multiple teams: The same individual submitting from or contributing to multiple teams.
  • Plagiarism: Submitting work that is substantially copied from another team without attribution.
  • External communication during evaluation: Agents may not make network calls outside the evaluation environment during official runs (except to allowed LLM API endpoints).

5. Submission Mechanics

  • Submissions are dockerized A2A-compatible agents (template provided in the starter kit).
  • Dev set: Unlimited submissions. Results appear on the live dev leaderboard.
  • Hidden test set: Two official evaluation rounds (Jun 14 and Jul 7). Each team gets one submission per round.
  • A submission is valid if it: (a) builds successfully from the Docker template, (b) responds to A2A protocol messages, and (c) completes evaluation within the time limit.
  • If a submission fails to build or run, the team receives a failure notification and may resubmit (dev set: immediately; test set: within a 24-hour grace window).
  • LLM API keys are provided through GitHub Secrets — no local computation is required for official evaluation.

6. Evaluation

  • Metric: Pass^3 — a task scores 1 only if solved correctly in all 3 independent trials.
  • Ranking: Teams are ranked by aggregate Pass^3 on the hidden test set (125 tasks).
  • Per-category scores (Base / Hallucination / Disambiguation) are reported alongside the aggregate.
  • Tiebreaking: In case of tied Pass^3, ties are broken by: (1) Pass^1 score, (2) submission time (earlier wins).
  • Variance: Each task is run 3 times with independent random seeds. The evaluation environment is deterministic given a seed; variance comes only from the LLM's non-determinism.
  • Reruns: Organizers reserve the right to rerun evaluations in case of infrastructure failures. Teams will be notified.

7. Dispute Resolution

  • Disputes must be submitted in writing to carbench-challenge@car-bench.org within 72 hours of the contested event.
  • Organizers will respond within 5 business days.
  • Organizers will adjudicate disputes concerning: evaluation correctness, rule violations, and eligibility questions.
  • Organizers will not adjudicate: subjective design disagreements, team-internal disputes, or matters outside the competition scope.
  • The organizers’ decision is final.

8. Code Release

  • Winners must release their code under a permissive open-source license (MIT, Apache 2.0, or equivalent) upon conclusion of the competition.
  • Code must be sufficient for independent reproduction of reported results.
  • Non-winning teams are encouraged (but not required) to release their code.

9. Technical Reports

  • A 4-page technical report (IJCAI format, excluding references) is required for award eligibility.
  • Deadline: July 14, 2026 (AoE).
  • Reports should describe the approach, design decisions, ablation results, and lessons learned.
  • Selected reports will be published as non-archival proceedings on the competition website.
  • Submission portal link will be added here when available.

10. Data Use

  • The CAR-bench dataset is released under the MIT License.
  • Participants may use the training and dev data for other research, provided they cite the original paper.
  • The hidden test set will not be released during the competition. Post-competition release will be announced separately.
  • Results and rankings will be made publicly available on the competition website after the final evaluation.
